# Lab 14 — Capstone: emoji to drop exec to R2 to GitHub

**Duration: 75 minutes**

This is the lab you have been building toward for two days. A GitHub issue comment
containing an emoji-encoded "capture 30" command triggers a chain of events that ends
with a pcap file landing as a signed R2 URL in a GitHub issue comment on the student's
command-queue issue, delivered by the Mango drop device you flashed and deployed in
Lab 12, routed through the Worker you have been incrementally building since Lab 07.

Nothing in this lab is new infrastructure. Everything uses endpoints and scripts that
already exist. The capstone is integration: making the full chain run reliably under a
time constraint.

<LabStep lab="lab14" id="encode" num={3} title="Encode the capture command with EmojiChef">

The command to encode is `capture 30` (30-second pcap). Use the EmojiChef encoder from
the Worker's test harness or from the operator console. The snippet below pads the
input to a multiple of 3 bytes with NUL before chunking, matching the deployed
`EmojiChefQuick` class. NUL pad bytes show up as trailing 🍅 (offset 0) in the
encoded output and the decoder strips them on read:

```sh
node -e "
const base = 0x1F345;       // 🍅 tomato
const bits = 6;
const text = 'capture 30';
const padLen = (3 - text.length % 3) % 3;
const padded = text + '\0'.repeat(padLen);
const bin = [...padded].map(c => c.charCodeAt(0).toString(2).padStart(8,'0')).join('');
const emojis = [];
for (let i = 0; i + bits <= bin.length; i += bits) {
    emojis.push(String.fromCodePoint(base + parseInt(bin.substr(i,bits),2)));
}
console.log(emojis.join(''));
// expected: 🍝🍻🍊🍵🍢🍌🍚🍷🍞🍗🍅🍸🍑🍅🍅🍅
"
```

Copy the emoji string. You will paste it into the GitHub issue in the next step.

To verify the round-trip, decode it back to `capture 30`:

```sh
node -e "
const base = 0x1F345;
const e = '🍝🍻🍊🍵🍢🍌🍚🍷🍞🍗🍅🍸🍑🍅🍅🍅';
const bin = [...e].map(c => (c.codePointAt(0) - base).toString(2).padStart(6,'0')).join('');
const decoded = (bin.match(/.{8}/g) || [])
    .map(b => String.fromCharCode(parseInt(b, 2))).join('').replace(/\0+\$/, '');
console.log(JSON.stringify(decoded));
// expected: \"capture 30\"
"
```

</LabStep>

<LabStep lab="lab14" id="dispatch" num={4} title="Post the emoji command to the GitHub issue">

The chatops endpoint for this capstone is `POST /v1/chatops/github`. The Worker
receives GitHub `issue_comment` webhook events, verifies the HMAC-SHA256 signature,
applies the allow-list and prefix gate, and decodes the emoji payload.

**Option A: post directly on GitHub.com (real webhook flow)**

Open your student command-queue issue (`https://github.com/${GITHUB_OWNER}/${GITHUB_REPO}/issues/${GITHUB_ISSUE_NUMBER}`)
and post a comment of the form:

```
@alpha <paste your emoji string here>
```

Replace `alpha` with your student slot. The GitHub webhook fires at the Worker automatically.

**Option B: simulate from the devcontainer (no webhook configured yet)**

This simulates the GitHub webhook POST directly:

```sh
EMOJI_CMD="<paste your emoji string here>"

# Build the payload GitHub would send for an issue_comment event
PAYLOAD=$(cat <<EOF
{
  "action": "created",
  "issue": {"number": ${GITHUB_ISSUE_NUMBER}},
  "comment": {
    "body": "@${STUDENT_SLOT} ${EMOJI_CMD}",
    "user": {"login": "${GITHUB_OWNER}", "type": "User"}
  },
  "repository": {
    "name": "${GITHUB_REPO}",
    "owner": {"login": "${GITHUB_OWNER}"}
  }
}
EOF
)

# Compute HMAC-SHA256 signature
SIG=$(printf '%s' "$PAYLOAD" | openssl dgst -sha256 -hmac "$GITHUB_WEBHOOK_SECRET" -hex | sed 's/.*= //')

JOB_RESPONSE=$(curl -sf \
    -X POST \
    -H "Content-Type: application/json" \
    -H "X-GitHub-Event: issue_comment" \
    -H "X-Hub-Signature-256: sha256=${SIG}" \
    -H "CF-Access-Client-Id: ${SERVICE_TOKEN_ID}" \
    -H "CF-Access-Client-Secret: ${SERVICE_TOKEN_SECRET}" \
    -d "$PAYLOAD" \
    "https://api.${DOMAIN}/v1/chatops/github")

echo "$JOB_RESPONSE"
JOB_ID=$(printf '%s' "$JOB_RESPONSE" | grep -o '"job_id":"[^"]*"' | sed 's/"job_id":"//;s/"//')
echo "JOB_ID=${JOB_ID}"
```

Expected response:

```json
{
  "decoded": "capture 30",
  "command": "capture",
  "args": ["30"],
  "job_id": "<uuid>",
  "status": "queued",
  "device_id": "drop-alpha"
}
```

</LabStep>

<LabStep lab="lab14" id="bridge" num={6} title="Operator bridge: dispatch to the Mango">

The Worker cannot initiate Tailscale connections. Workers are stateless edge functions
running on Cloudflare's network with no persistent state and no access to the tailnet.
The operator (you, in the devcontainer) is the physical bridge.

Read the job, extract the duration, and dispatch via `tailscale ssh`. Export the GitHub
variables so `run-capture.sh` can post the result comment:

```sh
DURATION=$(curl -sf \
    -H "CF-Access-Client-Id: ${SERVICE_TOKEN_ID}" \
    -H "CF-Access-Client-Secret: ${SERVICE_TOKEN_SECRET}" \
    "https://api.${DOMAIN}/v1/jobs/${JOB_ID}" \
    | grep -o '"args":\["[0-9]*"\]' | grep -o '[0-9]*')

DURATION="${DURATION:-30}"

tailscale ssh root@drop-${STUDENT_SLOT} \
    "GITHUB_TOKEN='${GITHUB_TOKEN}' \
     GITHUB_OWNER='${GITHUB_OWNER}' \
     GITHUB_REPO='${GITHUB_REPO}' \
     GITHUB_ISSUE_NUMBER='${GITHUB_ISSUE_NUMBER}' \
     STUDENT_SLOT='${STUDENT_SLOT}' \
     sh /tmp/run-capture.sh ${JOB_ID} ${DURATION} \
         https://api.${DOMAIN} ${SERVICE_TOKEN_ID} ${SERVICE_TOKEN_SECRET}"
```

The `run-capture.sh` script on the Mango will:

1. Run `tcpdump-mini -G ${DURATION} -W 1 -w /tmp/${JOB_ID}.pcap` (capture for N seconds).
2. POST to `/v1/artifacts/upload` to get a signed R2 PUT URL.
3. PUT the pcap file to R2 via the signed URL.
4. PATCH `/v1/jobs/${JOB_ID}/complete` to report the artifact_id and duration.
5. POST a result comment to the GitHub issue via the GitHub API.

Expected output streaming back over the SSH connection:

```
[run-capture.sh] starting tcpdump-mini for 30s, job=<uuid>
[run-capture.sh] capture complete, size=NNNN bytes, duration=30s
[run-capture.sh] minting upload URL...
[run-capture.sh] upload URL minted
[run-capture.sh] uploading pcap (NNNN bytes)...
[run-capture.sh] upload complete (HTTP 200)
[run-capture.sh] reporting completion...
[run-capture.sh] completion reported (HTTP 200)
[run-capture.sh] result posted to GitHub issue #1 (attempt 1)
[run-capture.sh] done; pcap removed from /tmp
```

</LabStep>

<LabStep lab="lab14" id="poll" num={7} title="Poll for completion and retrieve the download URL">

```sh
# Poll until status=complete (timeout 90s)
for i in $(seq 1 18); do
    RESULT=$(curl -sf \
        -H "CF-Access-Client-Id: ${SERVICE_TOKEN_ID}" \
        -H "CF-Access-Client-Secret: ${SERVICE_TOKEN_SECRET}" \
        "https://api.${DOMAIN}/v1/jobs/${JOB_ID}" 2>/dev/null)
    STATUS=$(printf '%s' "$RESULT" | grep -o '"status":"[^"]*"' | sed 's/"status":"//;s/"//')
    printf 'status=%s\n' "$STATUS"
    if [ "$STATUS" = "complete" ]; then
        printf '%s\n' "$RESULT"
        break
    fi
    sleep 5
done
```

Extract the download URL:

```sh
ARTIFACT_ID=$(printf '%s' "$RESULT" | grep -o '"artifact_id":"[^"]*"' | sed 's/"artifact_id":"//;s/"//')
DOWNLOAD_JSON=$(curl -sf \
    -H "CF-Access-Client-Id: ${SERVICE_TOKEN_ID}" \
    -H "CF-Access-Client-Secret: ${SERVICE_TOKEN_SECRET}" \
    "https://api.${DOMAIN}/v1/artifacts/${ARTIFACT_ID}")
DOWNLOAD_URL=$(printf '%s' "$DOWNLOAD_JSON" | grep -o '"download_url":"[^"]*"' | sed 's/"download_url":"//;s/"//')
echo "DOWNLOAD_URL=${DOWNLOAD_URL}"
```

</LabStep>

<LabStep lab="lab14" id="result" num={9} title="Confirm the result comment on the issue">

Open the command-queue issue and locate the result comment posted by `run-capture.sh`.
It will appear after the operator posts the emoji command. The comment body shape is:

```
[eplabs:result] @alpha capture complete
download: https://...r2.cloudflarestorage.com/...
artifact: <uuid>.pcap
duration: 30s
size: NNNN bytes
job: <uuid>
```

The `[eplabs:result]` sentinel at the start of the body is the loop-protection marker.
The Worker's `issue_comment` handler skips any incoming comment that begins with this
sentinel (and any comment from a Bot account) so the result post does not re-trigger
the pipeline.

You can also verify the comment was created via the GitHub API:

```sh
curl -sf \
    -H "Authorization: Bearer ${GITHUB_TOKEN}" \
    -H "Accept: application/vnd.github+json" \
    -H "X-GitHub-Api-Version: 2022-11-28" \
    "https://api.github.com/repos/${GITHUB_OWNER}/${GITHUB_REPO}/issues/${GITHUB_ISSUE_NUMBER}/comments" \
    | grep -A5 'eplabs:result'
```

</LabStep>

## Troubleshooting

<details>
<summary>EmojiChef decode returns garbage or fails</summary>

The emoji string must contain only codepoints in the range U+1F345 (🍅) to U+1F384 (🎄).
If you typed the emojis manually, verify them with:

```sh
node -e "const s='<your string>'; [...s].forEach(e => console.log(e, 'U+'+e.codePointAt(0).toString(16).toUpperCase()))"
```

Any codepoint outside the range will cause a decode error. Use the Node.js encoder from
Step 3 to generate the canonical string.

</details>

<details>
<summary>tailscale ssh fails to reach drop-${STUDENT_SLOT}</summary>

Run `tailscale status` in the devcontainer. If `drop-${STUDENT_SLOT}` is not listed,
the Mango may have lost its ExtRoot overlay (USB disconnected, or the Mango rebooted
without USB present). Re-plug the USB, then run:

```sh
ssh root@192.168.8.1 '/etc/init.d/tailscale restart'
sleep 10
tailscale status | grep drop-
```

</details>

<details>
<summary>run-capture.sh: "tcpdump-mini: command not found"</summary>

`tcpdump-mini` is in the NOR image (canonical package list from Lab 02). If it is missing,
the Mango firmware is not the sealed image from Lab 12. Flash the sealed image and retry.

</details>

<details>
<summary>pcap upload fails (curl error or HTTP 403)</summary>

The signed PUT URL from `/v1/artifacts/upload` has a 15-minute TTL. If more than 15 minutes
passed between minting the URL and the PUT, the URL is expired. Run `run-capture.sh` again
with the same job_id; it mints a fresh URL each time.

Also confirm the R2 binding is active: `wrangler r2 bucket list` should show `artifacts-bucket`.

</details>

<details>
<summary>GitHub comment POST fails (HTTP 401 or 403)</summary>

`GITHUB_TOKEN` must be a personal access token (or a fine-grained token) with at least
`repo` scope (or `issues: write` for fine-grained tokens) on the target repository.
Verify it is exported correctly in the environment before calling `tailscale ssh`.

Test the token independently:

```sh
curl -sf \
    -H "Authorization: Bearer ${GITHUB_TOKEN}" \
    -H "Accept: application/vnd.github+json" \
    "https://api.github.com/repos/${GITHUB_OWNER}/${GITHUB_REPO}/issues/${GITHUB_ISSUE_NUMBER}/comments" \
  | python3 -m json.tool | head -20
# Should list existing comments without error
```

</details>

<details>
<summary>PATCH /v1/jobs/&lt;id&gt;/complete returns 404</summary>

Check the Worker route for the `/v1/jobs/*/complete` path. A 404 from the Worker means
the path did not match any route case. Verify the Worker was deployed after Lab 14's
changes to `src/index.js` were merged in.

```sh
npx wrangler deploy --dry-run  # verify the Worker parses cleanly
npx wrangler deploy
```

</details>

<details>
<summary>Result comment does not appear; Worker re-triggers on its own post</summary>

The loop protection relies on `run-capture.sh` posting the comment directly (not via the
Worker). If you route the result through the Worker and the Worker posts the comment under
its own identity, ensure the Worker's `issue_comment` handler checks for the
`[eplabs:result]` prefix before processing. See `dispatch-flow.md` for the sentinel
contract.

</details>

## Primitives and drop-in substitutes

The result delivery channel in this lab is a GitHub issue comment. The same pattern
applies to other channels with minimal changes to `run-capture.sh`:

| Channel | Change required |
|---|---|
| **Discord** | `notify_discord()` is already in `run-capture.sh`, gated on `DISCORD_WEBHOOK_URL`. Set that env var to enable it as an extra channel. |
| **Slack** | Replace the GitHub comment POST with a POST to `https://hooks.slack.com/services/...` using Slack's incoming webhook JSON schema (`{"text": "..."}` or `{"blocks": [...]}`). |
| **Mattermost** | Same incoming webhook format as Slack; only the URL changes. |
| **Matrix** | POST to `https://<homeserver>/_matrix/client/v3/rooms/<room_id>/send/m.room.message` with a Bearer token and `{"msgtype": "m.text", "body": "..."}` body. |
| **Telegram** | POST to `https://api.telegram.org/bot<token>/sendMessage` with `{"chat_id": "<id>", "text": "..."}`. |
| **MQTT broker** | `mosquitto_pub -h <broker> -t eplabs/results -m "[eplabs:result] @${STUDENT_SLOT} ..."`. No HTTP required; useful for air-gapped ops nets. |
| **Custom HTTPS webhook** | Any HTTP endpoint that accepts a JSON POST. Adjust the curl command and authentication header as needed. |

For each substitute, the `[eplabs:result]` sentinel prefix and the `download:` line
on the second line of the body should be preserved so that any automated consumers
can parse the result consistently.
